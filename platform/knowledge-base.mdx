### Connecting a Database

<Steps>
  <Step title="Click Add Connection">
    Navigate to Knowledge Base → Data Sources → Add Connection
  </Step>
  <Step title="Select Database Type">
    Choose from the supported database types

    <Frame caption="Add a new data source connection">
      <img src="/images/add-datasource-form.png" alt="Add Data Source Form" />
    </Frame>
  </Step>
  <Step title="Enter Connection Details">
    - Host/Server address
    - Port number
    - Database name
    - Username and password (read-only)
  </Step>
  <Step title="Test Connection">
    Verify connectivity before saving
  </Step>
  <Step title="Automatic Semantic Modeling">
    Superatom runs a five-stage automated analysis to build a semantic model from your data
  </Step>
</Steps>

### Automated Semantic Modeling

When a data source is connected, Superatom automatically builds a semantic model through five stages:

<Steps>
  <Step title="Schema Analysis">
    Reads database structure: tables, columns, data types, relationships, foreign keys, indexes
  </Step>
  <Step title="Sample Data Extraction">
    Examines representative samples to understand actual data characteristics, edge cases, and formats
  </Step>
  <Step title="Statistical Profiling">
    Calculates distinct values, distributions, frequencies, ratios, and patterns across columns
  </Step>
  <Step title="Domain Classification">
    Recognizes industry context (supply chain, retail, finance, healthcare) and applies domain-specific mappings
  </Step>
  <Step title="Question Generation">
    Proposes common, diagnostic, and exploratory questions that can be asked of this data
  </Step>
</Steps>

<Note>
Traditional semantic modeling requires specialized consultants and takes 3-6 months. This automated process completes in approximately 2 days and improves continuously as the system is used.
</Note>

### Example: Overstock Definition

```yaml
title: "Overstock"
type: "classification"

description: |
  Overstock items are items where the On Hand Cash Value
  exceeds the Cost of Goods Sold (COGS) over a rolling period.
  The rolling period varies by product type.

rules:
  - "Structural products (ProductType = 'Structural'): Rolling 90 days COGS > $0"
  - "Decking products (MajorMinorCode = '0103DEC'): Rolling 180 days COGS > $0"
  - "Specialty products: Rolling 120 days COGS > $0"

key_columns:
  - "FactInventoryDaily.OnHandInventory"
  - "FactSalesInvoice.AverageCostExtended"
  - "DimItem.ProductType"
  - "DimItem.MajorMinorCode"

formula: "OnHandInventory - (Rolling Period COGS)"
```
