## Creating a Dashboard

<Steps>
  <Step title="Navigate to Dashboards">
    Click **Dashboards** in the sidebar under BUILD
  </Step>
  <Step title="Click Create Dashboard">
    Start with a blank canvas
  </Step>
  <Step title="Add Components">
    - Ask questions in natural language, or
    - Add from bookmarks, or
    - Drag from component library
  </Step>
  <Step title="Arrange Layout">
    Drag and resize components to create your layout
  </Step>
  <Step title="Save & Share">
    Name your dashboard and set permissions
  </Step>
</Steps>

## Dashboard Components

### From Natural Language

Type a question directly in the dashboard:

1. Click **+ Add Component**
2. Type your question: *"Revenue by region this quarter"*
3. Superatom generates the visualization
4. Resize and position as needed

### From Bookmarks

Reuse visualizations you've already created:

1. Click **+ Add from Bookmarks**
2. Select from your saved visualizations
3. Component added with same configuration

## Interactive Features

### Filtering System

Dashboards support a layered filtering system that gives users precise control over what data is displayed.

#### Global Filters

Affect all tiles simultaneously:

- **Date Range** — Filter entire dashboard by time period
- **Region** — Show data for specific regions
- **Product** — Focus on product categories
- **Custom** — Any dimension in your data

Global filters are auto-derived from the data dimensions present across tiles. When you add tiles that share common dimensions (like region or date), those dimensions automatically become available as global filters.

#### Tile-Level Filters

Individual tiles can have their own filter overrides that narrow or modify the global filter scope. For example, a global filter set to "All Regions" can be overridden on a specific tile to show only "Northeast" for a focused comparison.

#### Cross-Filtering

Click a data point in one tile to filter related tiles:

- Click "Northeast" on a regional bar chart — all other tiles filter to Northeast data
- Click "Q3" on a timeline — all tiles narrow to Q3
- Click a product category — see its impact across all metrics

Cross-filtering works automatically when tiles share common dimensions.

#### Filter Persistence

Filter selections are saved with the dashboard. When users return, they see their last-applied filters. Shared dashboards preserve the creator's default filters, while each viewer can apply their own overrides without affecting others.

#### URL Parameters

Filters can be set via URL parameters, enabling direct links to specific filtered views:

```
/dashboards/executive-overview?region=Northeast&period=Q3-2025
```

This is useful for sending pre-filtered dashboard links in reports, emails, or Slack alerts.

### Drill-Down

Click any data point to:
- See underlying details
- Open related visualizations
- Ask follow-up questions

### Auto-Refresh

Keep dashboards current:

| Interval | Use Case |
|----------|----------|
| Real-time | Operations centers |
| 5 minutes | Active monitoring |
| 15 minutes | Standard dashboards |
| Manual | Point-in-time analysis |

## Sharing & Permissions

### Access Levels

| Level | Capabilities |
|-------|-------------|
| **View** | See the dashboard |
| **Edit** | Modify components and layout |
| **Admin** | Manage sharing and delete |

### Sharing Options

- **Specific Users** — By email or username
- **Roles** — All users with a role
- **Public** — Anyone with the link (optional)

## Best Practices

<AccordionGroup>
  <Accordion title="Keep It Focused" icon="bullseye">
    One dashboard = one purpose. Don't try to show everything.
  </Accordion>
  <Accordion title="Prioritize Top-Left" icon="arrow-up-left">
    Most important metrics go top-left (where eyes start).
  </Accordion>
  <Accordion title="Use Consistent Sizing" icon="ruler">
    Similar components should be similar sizes.
