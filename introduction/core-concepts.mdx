## The Decision Loop

Superatom implements a continuous decision-making loop:

```mermaid
flowchart LR
    A["UNDERSTAND"] --> B["ANALYZE"]
    B --> C["SUGGEST"]
    C --> D["VERIFY"]
    D --> E["EXECUTE"]
    E --> F["OBSERVE"]
    F --> A

    style A fill:#0D9373,color:#fff
    style B fill:#0D9373,color:#fff
    style C fill:#0D9373,color:#fff
    style D fill:#07C983,color:#fff
    style E fill:#07C983,color:#fff
    style F fill:#07C983,color:#fff
```

| Stage | What Happens |
|-------|--------------|
| **Understand** | Connect data, build semantic model |
| **Analyze** | AI finds patterns, anomalies, insights |
| **Suggest** | Recommend actions to address findings |
| **Verify** | Human reviews and approves |
| **Execute** | Action committed to systems |
| **Observe** | Track outcomes, learn from results |

## Tribal Knowledge

Organization-specific knowledge captured at three levels:

```mermaid
flowchart TD
    QUESTION["User Question"] --> ENGINE["Knowledge Engine"]
    ENGINE --> G["🌍 GLOBAL"] & U["👤 USER"] & Q["❓ QUERY"]
    G & U & Q --> ANSWER["Contextualized Answer"]

    style ENGINE fill:#0D9373,color:#fff
```

| Level | Example | Scope |
|-------|---------|-------|
| **Global** | "Sales exclude returns" | All users, all queries |
| **User** | "Show only West Coast data" | Specific user |
| **Query** | "Deadstock = no sales in 12 months" | Specific question type |

## Knowledge Nodes

Building blocks of organizational knowledge:

```yaml
knowledge_node:
  title: "Overstock Definition"
  type: query

  description: |
    Items where inventory exceeds 90 days
    of average sales velocity

  rules:
    - "Calculate days_supply = qty / avg_daily_sales"
    - "Overstock when days_supply > 90"

  applies_to:
    - "inventory queries"
    - "overstock analysis"
```

Knowledge nodes guide how the AI interprets and answers questions.

## Coding Agents

AI agents that understand business context:

```mermaid
flowchart LR
    Q["Complex Question"] --> AGENT["Coding Agent"]

    AGENT --> THINK["Think"]
    THINK --> ACT["Act"]
    ACT --> OBSERVE["Observe"]
    OBSERVE --> DECIDE{"Done?"}
    DECIDE -->|No| THINK
    DECIDE -->|Yes| ANSWER["Answer"]

    style AGENT fill:#0D9373,color:#fff
```

Agents can:
- Break complex questions into sub-tasks
- Generate and execute multiple queries
- Iterate until they find the answer
- Explain their reasoning

## Dry-Run Execution

Safe action execution with human oversight:

```mermaid
flowchart LR
    AI["AI Suggests Action"] --> SIM["Simulate"]
    SIM --> REVIEW["Human Review"]
    REVIEW --> APPROVE{"Approved?"}
    APPROVE -->|Yes| EXECUTE["Execute"]
    APPROVE -->|No| MODIFY["Modify/Reject"]

    style SIM fill:#f59e0b,color:#fff
    style REVIEW fill:#0D9373,color:#fff
```

Actions are simulated first, showing what WOULD happen before committing.

## Real-Time Streaming

Responses appear as they're generated:

- **Text** streams character by character
- **Charts** build progressively
- **Tables** populate row by row
- **Follow-ups** appear after main content

This creates a more engaging, responsive experience than waiting for complete results.

## Multi-LLM Support

No single AI vendor dependency:

| Provider | When Used |
|----------|-----------|
| Anthropic Claude | Complex reasoning tasks |
| OpenAI GPT | General purpose queries |
| Google Gemini | Fast processing needs |
| Groq | Real-time, high-volume |

Switch providers based on task, cost, or preference.
