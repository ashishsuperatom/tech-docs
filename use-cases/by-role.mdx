## Overview

Superatom adapts to how different people work. Executives get strategic KPI summaries. Operations managers get real-time monitoring and exception alerts. Analysts get deep exploration tools. Field workers get task-specific mobile interfaces. Each role sees the same trusted data through the lens that matches their needs.

---

## Executive Leadership

**Primary use:** Strategic oversight, KPI monitoring, ad-hoc investigation

<Tabs>
  <Tab title="Morning Routine">
    Receive an automated daily KPI summary via email or mobile push notification. Review key metrics at a glance: revenue, margin, cash position, and operational metrics.
  </Tab>
  <Tab title="During Meetings">
    Ask follow-up questions directly. For example: *"Why is the margin lower than last week?"* Superatom provides an immediate root cause analysis with visualization -- no analyst needed.
  </Tab>
  <Tab title="Strategic Planning">
    Run cross-source analyses on demand. For example: *"Show me the correlation between marketing spend and customer acquisition over the past 12 months by region"* -- combining marketing and CRM data instantly.
  </Tab>
</Tabs>

<AccordionGroup>
  <Accordion title="Dashboard Example">
    Executive Overview with revenue, margin, cash flow, customer metrics, and operational KPIs. Auto-refreshing with drill-down enabled. Accessible from desktop and mobile.
  </Accordion>
</AccordionGroup>

## Operations Managers

**Primary use:** Daily operational monitoring, exception management, resource allocation

<Tabs>
  <Tab title="Real-Time Monitoring">
    Dashboard showing current operational status -- inventory levels, order fulfillment rates, production status, and workforce allocation. Updates in real time.
  </Tab>
  <Tab title="Exception Handling">
    Receive triggered alerts when metrics breach thresholds. Each alert includes root cause analysis and suggested actions. Review and act from any device.
  </Tab>
  <Tab title="Resource Decisions">
    Run scenario analyses on demand. For example: *"What's the impact if we shift 3 workers from Line A to Line B?"* Superatom models the outcome based on current production rates and demand.
  </Tab>
</Tabs>

<AccordionGroup>
  <Accordion title="Workflow Example">
    Triggered alert when order fulfillment rate drops below 95% leads to an automatic root cause investigation. The analysis is sent to the operations manager, who reviews and takes action from their mobile device.
  </Accordion>
</AccordionGroup>

## Data Analysts

**Primary use:** Deep analysis, report creation, data exploration

<Tabs>
  <Tab title="Exploratory Analysis">
    Use the Chat Agent for complex, multi-step investigations. Chain questions to drill deeper. Bookmark intermediate results for later reference.
  </Tab>
  <Tab title="Report Building">
    Create scheduled reports for stakeholders. Define the analysis, set the schedule, and configure delivery. Reports update automatically with fresh data.
  </Tab>
  <Tab title="Knowledge Curation">
    Define tribal knowledge nodes based on domain expertise. Set business rules that improve accuracy for all users across the organization.
  </Tab>
</Tabs>

<Note>
Analysts see the full SQL behind every answer. They can verify the logic, suggest corrections, and export data for additional analysis in their preferred tools.
</Note>

## Field Workers

**Primary use:** Task-specific information and actions from mobile devices

<Tabs>
  <Tab title="Warehouse Workers">
    Check inventory levels, record cycle counts, and flag discrepancies -- all from a mobile device with barcode scanning capability.
  </Tab>
  <Tab title="Sales Representatives">
    View customer order history, check product availability, and place orders during customer visits. All data is available in real time.
  </Tab>
  <Tab title="Maintenance Technicians">
    Access equipment history, log maintenance activities, and view work orders. Receive push notifications for urgent maintenance tasks.
  </Tab>
</Tabs>

<AccordionGroup>
  <Accordion title="Mobile Features">
    - **Role-based views** -- field workers see only what is relevant to their job
    - **Offline mode** -- cached data is available without connectivity; actions are queued and synced when reconnected
    - **Push notifications** -- time-sensitive alerts are delivered instantly to mobile devices
  </Accordion>
</AccordionGroup>
