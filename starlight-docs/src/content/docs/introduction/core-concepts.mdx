## The Decision Loop

Superatom implements a continuous decision-making loop:

<DecisionLoopDiagram client:load />

| Stage | What Happens |
|-------|--------------|
| **Understand** | Connect data, build semantic model |
| **Analyze** | AI finds patterns, anomalies, insights |
| **Suggest** | Recommend actions to address findings |
| **Verify** | Human reviews and approves |
| **Execute** | Action committed to systems |
| **Observe** | Track outcomes, learn from results |

## Knowledge Nodes

Building blocks of organizational knowledge:

```yaml
knowledge_node:
  title: "Overstock Definition"
  type: query

  description: |
    Items where inventory exceeds 90 days
    of average sales velocity

  rules:
    - "Calculate days_supply = qty / avg_daily_sales"
    - "Overstock when days_supply > 90"

  applies_to:
    - "inventory queries"
    - "overstock analysis"
```

Knowledge nodes guide how the AI interprets and answers questions.

## Real-Time Streaming

Responses appear as they're generated:

- **Text** streams character by character
- **Charts** build progressively
- **Tables** populate row by row
- **Follow-ups** appear after main content

This creates a more engaging, responsive experience than waiting for complete results.

## Multi-LLM Support

No single AI vendor dependency:

| Provider | When Used |
|----------|-----------|
| Anthropic Claude | Complex reasoning tasks |
| OpenAI GPT | General purpose queries |
| Google Gemini | Fast processing needs |
| Groq | Real-time, high-volume |

Switch providers based on task, cost, or preference.
