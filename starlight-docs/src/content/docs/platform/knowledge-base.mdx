### Connecting a Database

1. **Click Add Connection**

   Navigate to Knowledge Base → Data Sources → Add Connection

  1. **Select Database Type**

   Choose from the supported database types

  1. **Enter Connection Details**

   - Host/Server address
    - Port number
    - Database name
    - Username and password

  1. **Test Connection**

   Verify connectivity before saving

  1. **Run Analysis**

   Superatom analyzes the schema and builds a semantic model

## Knowledge Nodes

Create and manage knowledge nodes that guide AI analysis:

  ![Knowledge Nodes List](/images/knowledge-nodes-list.png)

### Node Types

| Type | Purpose |
|------|---------|
| **Global** | Organization-wide rules and definitions |
| **User** | Role-specific or personal preferences |
| **Query** | Definitions for specific questions |

### Creating a Knowledge Node

  ![Knowledge Node Detail](/images/knowledge-node-detail.png)

Each node includes:

- **Title** — Clear, descriptive name
- **Description** — What this knowledge represents
- **Rules** — Specific conditions and calculations
- **Key Columns** — Data fields involved
- **Formula** — How to calculate (if applicable)

### Example: Overstock Definition

```yaml
title: "Overstock"
type: "classification"

description: |
  Overstock items are items where the On Hand Cash Value
  exceeds the Cost of Goods Sold (COGS) over a rolling period.
  The rolling period varies by product type.

rules:
  - "Structural products (ProductType = 'Structural'): Rolling 90 days COGS > $0"
  - "Decking products (MajorMinorCode = '0103DEC'): Rolling 180 days COGS > $0"
  - "Specialty products: Rolling 120 days COGS > $0"

key_columns:
  - "FactInventoryDaily.OnHandInventory"
  - "FactSalesInvoice.AverageCostExtended"
  - "DimItem.ProductType"
  - "DimItem.MajorMinorCode"

formula: "OnHandInventory - (Rolling Period COGS)"
```

## Data Model

View the semantic model Superatom has built:

### What's Included

- **Tables** — All discovered tables and their purposes
- **Columns** — Field-level details with data types
- **Relationships** — Foreign keys and joins
- **Statistics** — Row counts, distributions, patterns

### Using the Data Model

The data model powers:
- **Natural language understanding** — Mapping questions to tables
- **Query generation** — Building correct SQL
- **Visualization selection** — Choosing appropriate charts
- **Tribal knowledge application** — Applying org-specific rules
